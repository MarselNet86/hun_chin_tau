# HunChinTau

HunChinTau — платформа для просмотра аниме с современным интерфейсом, системой закладок и персонализированными рекомендациями.

![Превью проекта](screenshot.jpg)

## Особенности

- 🎬 **Каталог аниме** с фильтрами по жанрам, статусу, году, озвучке и рейтингу
- 🔖 **Закладки** с возможностью отслеживания статуса просмотра
- 🎥 **Встроенный видеоплеер** с поддержкой перемотки и горячих клавиш
- 🔍 **Поиск аниме** с автодополнением
- 👤 **Профиль пользователя** с историей закладок
- 🎨 **Минималистичный дизайн** без лишних элементов
- 📱 **Адаптивный интерфейс** для всех устройств

## Технологии

- **Backend:** Django 5.x, Python 3.10+
- **Frontend:** HTML5, CSS3, JavaScript (Vanilla)
- **Видеоплеер:** Plyr
- **3D эффекты:** Three.js
- **База данных:** SQLite (по умолчанию), PostgreSQL (рекомендуется для production)

## Требования

- Python 3.10 или выше
- pip (менеджер пакетов Python)
- FFmpeg (для обработки видео, опционально)

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd hun_chin_tau
```

### 2. Создание виртуального окружения

```bash
# macOS/Linux
python3 -m venv .venv
source .venv/bin/activate

# Windows
python -m venv .venv
.venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

Если файл `requirements.txt` отсутствует, установите основные пакеты:

```bash
pip install django pillow
```

### 4. Применение миграций

```bash
python manage.py migrate
```

### 5. Создание суперпользователя

```bash
python manage.py createsuperuser
```

### 6. Запуск сервера разработки

```bash
python manage.py runserver
```

Откройте браузер и перейдите по адресу: **http://127.0.0.1:8000/**

## Настройка

### Переменные окружения (опционально)

Создайте файл `.env` в корне проекта:

```env
DEBUG=True
SECRET_KEY=your-secret-key-here
ALLOWED_HOSTS=localhost,127.0.0.1

# База данных (для PostgreSQL)
DATABASE_URL=postgresql://user:password@localhost:5432/hun_chin_tau

# Медиа файлы
MEDIA_URL=/media/
MEDIA_ROOT=/path/to/media/root
```

### Настройка медиа файлов

Для загрузки изображений и видео убедитесь, что директории существуют и доступны для записи:

```bash
mkdir -p media/anime/posters
mkdir -p media/anime/covers
mkdir -p media/videos/episodes
mkdir -p media/avatars
```

### Добавление аниме

1. Войдите в админ-панель: **http://127.0.0.1:8000/admin/**
2. Добавьте жанры, студии, теги и озвучки
3. Создайте аниме:
   - Заполните названия (русское, английское, оригинальное)
   - Добавьте описание, постер и обложку
   - Выберите жанры, теги, студии
   - Укажите статус, год, рейтинг
4. **Добавьте сезоны** (внизу страницы аниме):
   - Нажмите "Добавить сезон"
   - Укажите номер сезона и название (если есть)
   - Сохраните
5. **Добавьте эпизоды** (внутри сезона):
   - Выберите сезон
   - Добавьте эпизоды с номерами и названиями
   - Для каждого эпизода укажите источник видео:
     - **Прямое видео** — загрузите файл или укажите URL
     - **Iframe** — вставьте ссылку на внешний плеер (Anitype, Kodik)
6. Добавьте озвучку и субтитры (опционально)

## Приложения

### anime
- Главная страница с Hero-секцией
- Страница аниме с описанием и эпизодами
- Поиск аниме

### catalog
- Расширенный каталог с фильтрами
- Сортировка по различным параметрам
- Фильтрация по жанрам, статусу, году, озвучке, рейтингу

### episodes
- Модели: Episode, Season, VoiceActor, PlayerSource
- Управление эпизодами и сезонами
- Источники видео (прямые файлы, iframe)
- **Разделение эпизодов по сезонам**

### player
- Видеоплеер на базе Plyr
- Перемотка, горячие клавиши
- Навигация между эпизодами

### users
- Регистрация, вход, выход
- Профиль пользователя
- Закладки с статусами (Смотрю, Просмотрено, В планах, Отложено, Брошено)

## Горячие клавиши (в плеере)

| Клавиша | Действие |
|---------|----------|
| `Пробел` | Пауза/Воспроизведение |
| `←` | Перемотка назад на 10 сек |
| `→` | Перемотка вперёд на 10 сек |
| `0-9` | Перемотка на 0%-90% видео |

## Production развёртывание

### 1. Настройка веб-сервера

Для production рекомендуется использовать **Gunicorn** + **Nginx**:

```bash
pip install gunicorn
gunicorn hun_chin_tau.wsgi:application --bind 0.0.0.0:8000
```

### 2. Сбор статических файлов

```bash
python manage.py collectstatic
```

### 3. Настройка Nginx

Пример конфигурации:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /static/ {
        alias /path/to/static/root/;
    }

    location /media/ {
        alias /path/to/media/root/;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 4. База данных

Для production рекомендуется использовать **PostgreSQL**:

```bash
pip install psycopg2-binary
```

В `settings.py` настройте подключение к PostgreSQL.

```bash

python manage.py dumpdata > backup.json

# Медиа файлы
tar -czf media-backup.tar.gz media/
```

### Восстановление из резервной копии

```bash
python manage.py loaddata backup.json
```
